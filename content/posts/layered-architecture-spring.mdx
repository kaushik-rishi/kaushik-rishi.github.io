### Outline
- JAR vs WAR
  - jar: standalone java application, only java, libraries, microservices (java archive)
  - war: web archive, jsp + html + css + java
- Layered Architecture
  - Controller Layer - `@Controller`, `@RestController`
    - API route frontend
    - Mapping of API Parameters to `RequestDTO`
  - Service Layer - `@Service`
    - Business logic
    - Mapping of DB data to `ResponseDTO`
    - Example, Create `Entity` and give to repository layer to insert
  - Repository Layer - `@Repository`
    - Interacts with DB
- Other packages / modules
  - DTO (data transfer object)
    - `RequestDTO` - map the data coming from external client
    - `ResponseDTO` - map the data being sent out from the server
  - Utility (common methods used in services)
  - Entity - `@Entity`
    - POJO's
    - Direct Representation of the Table
  - Configuration - `application.properties`
